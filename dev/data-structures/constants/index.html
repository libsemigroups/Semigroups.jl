Constants

This page describes the constants used in Semigroups.jl.

Constant Values

Semigroups.UNDEFINED — Constant
UNDEFINED

Value for something undefined.

This variable is used to indicate that a value is undefined. UNDEFINED is comparable with any value via == and != but not via < or >.

Examples

using Semigroups

p = PPerm([2, UNDEFINED, 1])
p[2]  # UNDEFINED
p[2] == UNDEFINED  # true
p[1] == UNDEFINED  # false
source
Semigroups.POSITIVE_INFINITY — Constant
POSITIVE_INFINITY

Represents positive infinity in libsemigroups. Can be compared with integers and NEGATIVE_INFINITY using <, >, ==, !=. Converts to max-1 of the target integer type.

source
Semigroups.NEGATIVE_INFINITY — Constant
NEGATIVE_INFINITY

Represents negative infinity in libsemigroups. Can be compared with signed integers and POSITIVE_INFINITY using <, >, ==, !=. Converts to the minimum value of the target signed integer type.

source
Semigroups.LIMIT_MAX — Constant
LIMIT_MAX

Represents the maximum limit value in libsemigroups. Converts to max-2 of the target integer type.

source

Predicate Functions

Semigroups.is_undefined — Function
is_undefined(x) -> Bool

Return true if x is UNDEFINED, false otherwise.

Examples

using Semigroups

p = PPerm([2, UNDEFINED, 1])
is_undefined(p[2])  # true
is_undefined(p[1])  # false

Ternary Logic (tril)

Semigroups.tril — Type
tril

Ternary logic type representing true, false, or unknown values. Use tril_TRUE, tril_FALSE, and tril_unknown for the possible values.

source

tril_to_bool(t::tril) -> Union{Bool, Nothing}
