The Paths type

This page documents the type Paths, a stateful range over paths in a WordGraph. A Paths object pins its source word graph for garbage collection and yields paths as Vector{Int} of 1-based edge labels via the standard Julia iteration protocol or the manual get / next! / at_end interface.

v1 limitation

Paths is bound for WordGraph<uint32_t> only; other Node types follow when consumers need them.

Semigroups.Paths — Type
Paths

Type for a stateful range over paths in a WordGraph.

A Paths object wraps libsemigroups' Paths<uint32_t> together with a reference to the source WordGraph that pins it for garbage collection. The C++ object stores only a raw pointer to the word graph, so keeping the Julia wrapper alive is what keeps the underlying graph alive.

Configure with source!, target!, min!, max!, order!, then iterate with the standard Julia iteration protocol (for w in p, collect(p)) or via the manual interface (Base.get, next!, at_end, Base.count).

Lifetime when constructed from a borrowed reference

A Paths may also be constructed from a WordGraph reference borrowed from another object - for example, the result of current_word_graph or word_graph on a ToddCoxeter instance, both of which return a CxxWrap.CxxBaseRef{WordGraph} viewing memory owned by the source object. In that case the Paths wrapper does not pin the underlying owner; the borrowed reference does not extend its lifetime. The caller must keep the source object (e.g. the ToddCoxeter) alive for as long as the Paths is in use.

Example

julia> using Semigroups

julia> g = WordGraph(3, 2);

julia> target!(g, 1, 1, 2); target!(g, 1, 2, 3); target!(g, 2, 1, 3);

julia> p = paths(g; source = 1, max = 3, order = ORDER_SHORTLEX);

julia> collect(p)
4-element Vector{Vector{Int64}}:
 []
 [1]
 [2]
 [1, 1]

Semigroups.Paths — Method
Paths(g::WordGraph) -> Paths

Construct a new Paths over g.

The new range has source and target UNDEFINED, minimum length 0, maximum length POSITIVE_INFINITY, and order ORDER_SHORTLEX. At least the source must be set (via source!) before the range can be iterated.

Arguments

  • g::WordGraph: the word graph.

See also paths.

Semigroups.paths — Method
paths(g::WordGraph;
      source = UNDEFINED,
      target = UNDEFINED,
      min::Integer = 0,
      max = POSITIVE_INFINITY,
      order::Order = ORDER_SHORTLEX) -> Paths

Construct a Paths over g configured by keyword arguments.

This is a convenience factory equivalent to constructing a Paths(g) and chaining the relevant !-suffixed setters. Argument names mirror the underlying setting names (source!, target!, min!, max!, order!).

Arguments

  • g::WordGraph: the word graph.

Keywords

  • source: the source node (1-based Integer) or UNDEFINED (default).
  • target: the target node (1-based Integer) or UNDEFINED (default), meaning "any reachable target".
  • min::Integer: minimum path length (default 0).
  • max: maximum path length (Integer or POSITIVE_INFINITY — the default).
  • order::Order: ORDER_SHORTLEX (default) or ORDER_LEX.

Example

julia> using Semigroups

julia> g = WordGraph(3, 2);

julia> target!(g, 1, 1, 2); target!(g, 2, 1, 3);

julia> count(paths(g; source = 1, max = 5))
3
Lifetime when `g` is a borrowed reference

g may also be a CxxWrap.CxxBaseRef{WordGraph} borrowed from another object - for example, the result of current_word_graph or word_graph on a ToddCoxeter instance. In that case the returned Paths does not pin the source object; the caller must keep that owner (e.g. the ToddCoxeter) alive for as long as the Paths is in use. See the note on Paths.

Semigroups.init! — Method
init!(p::Paths, g::WordGraph) -> Paths

Reinitialize p to range over g, resetting all settings to their defaults.

After this call, p is in the same state as a freshly constructed Paths(g): source and target are UNDEFINED, minimum length 0, maximum length POSITIVE_INFINITY, and order ORDER_SHORTLEX. The wrapper's GC pin is updated so the new word graph is kept alive.

Arguments

  • g::WordGraph: the new word graph.

See also Paths.

Semigroups.throw_if_source_undefined — Method
throw_if_source_undefined(p::Paths) -> nothing

Throw if the source node of p has not been set.

This function is the Julia mirror of libsemigroups' Paths::throw_if_source_undefined(). It is called automatically by Base.get, next!, at_end, and Base.count since the underlying C++ implementation does not internally guard those operations.

Throws

  • LibsemigroupsError: if source(p) is UNDEFINED.

Base.count — Method
Base.count(p::Paths) -> Union{Int, PositiveInfinityType}

Return the number of paths in the range.

If the range is infinite (cyclic graph with no upper bound on length), returns POSITIVE_INFINITY. Always throws if source(p) === UNDEFINED; the source must be set before counting.

Throws

  • LibsemigroupsError: if source(p) is UNDEFINED.
